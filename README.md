# Тестовое задание для Next.js разработчика

## О проекте
Данное тестовое задание демонстрирует создание простого веб-приложения на Next.js с использованием App Router, Bun и Redux. Основная цель – реализовать авторизацию, поддержку i18next для переводов и адаптивный дизайн.

---


## Как запустить проект

```bash
bun install
bun dev
```

## Что реализовано

### 1. Общие требования
- Использован **Next.js (App Router)**.
- Проект запускается через **Bun**.
- Реализована локализация через **i18next** (доступны языки `ru` и `en`).
- Поддерживается адаптивность интерфейса.

### 2. Авторизация
- Пользователь может **зарегистрироваться и войти** с любым email и паролем.
- Пароль хранится в виде **SHA256-хеша**.
- После авторизации состояние сохраняется через **Redux + redux-persist**, поэтому при обновлении страницы авторизация сохраняется.
- Страница `/about` доступна только **авторизованным пользователям**.
- Страница `/register` доступна только **неавторизованным пользователям**.

### 3. Страницы 
- Отображают **демо-описание проекта**.
- Содержат **2-3 блока информации**, демонстрируя структуру страницы.

### 4. Хедер
- Общий для всех страниц.
- Содержит ссылки: **Home**, **About**, **Login / Logout** (в зависимости от авторизации).
- Отображает **email авторизованного пользователя**.
- Включает **переключатель языка** через i18next.

---

## Технологии
- Next.js 16 (App Router)
- React 19
- Redux + Redux Toolkit + redux-persist
- i18next + react-i18next
- TailwindCSS
- Jest + React Testing Library
- SHA256 для хранения пароля
